# 编辑模式修复计划

## 问题分析

根据用户需求，需要修复以下三个问题：

### 问题1：进入编辑模式不再以列表展现
- 当前行为：`toggleEditMode()` 函数会强制切换到列表模式
- 期望行为：进入编辑模式时保持当前视图模式（卡片/列表）

### 问题2：卡片模式下编辑功能
- 当前行为：编辑模式只在列表模式下显示操作按钮
- 期望行为：卡片模式下进入编辑模式后，在图片右上角显示编辑按钮
- 点击编辑按钮后弹出编辑弹窗，其中：
  - 类别和图片只显示不可修改
  - 属性高亮显示已选值，可修改属性值
  - 弹出框左下角有删除按钮（带确认弹窗）
  - 右下角是取消和保存按钮

### 问题3：列表模式下编辑和删除按钮
- 当前行为：编辑按钮和删除按钮在 hover 时显示
- 期望行为：编辑模式下始终显示编辑和删除按钮，编辑按钮更明显，中间有间隔

## 实现方案

### 1. 修改 `toggleEditMode()` 函数
- 移除强制切换到列表模式的逻辑
- 只切换编辑状态，保持当前视图模式

### 2. 修改卡片渲染逻辑 (`renderItems()` 函数)
- 编辑模式下，卡片模式也显示编辑和删除按钮
- 编辑按钮显示在图片右上角

### 3. 创建编辑弹窗组件
- 复用现有 `itemModal` 模态框
- 添加删除按钮到模态框底部

### 4. 修改列表渲染逻辑 (`updateListView()` 函数)
- 编辑模式下始终显示编辑和删除按钮
- 调整按钮样式，使编辑按钮更明显

### 5. 添加删除确认弹窗
- 创建删除确认模态框
- 确认后删除物品

## 代码改动概述

1. **toggleEditMode 函数**：移除 `setViewMode('list')` 调用
2. **renderItems 函数**：在卡片模式下也渲染编辑和删除按钮
3. **updateListView 函数**：调整按钮样式和布局
4. **editItem 函数**：设置类别和图片为只读状态
5. **HTML 结构**：添加删除确认模态框和删除按钮
