# 批量编辑属性功能修复计划

## 需求分析

根据用户需求，需要修复以下6个问题：

1. **批量编辑属性按钮**：只有选择的物品均为同一类别才可用，否则禁用，按钮名称改为"同类属性编辑"
2. **属性列表显示**：属性应该从所选物品类别的模板中读取，样式和添加物品时一致，默认展开所有属性值
3. **选中样式**：选中属性值时整行高亮黄色，末尾显示对勾
4. **添加属性逻辑**：如果物品已有该属性则跳过，否则添加；只有点击按钮时才保存
5. **移除属性逻辑**：如果物品没有该属性值则跳过，否则移除；只有点击按钮时才保存
6. **校验**：保存前校验是否至少选中一条属性值，否则提示错误

## 代码分析

### 关键文件

- `frontend/html/index.html` - 前端主文件，包含批量编辑模态框和相关逻辑
- `backend/routes/items.py` - 物品相关API，包含批量属性操作接口

### 当前实现问题

1. 当前按钮名称为"批量编辑属性"，未检查物品类别是否相同
2. 属性选择器使用全局属性列表`attributes`，而非类别模板属性
3. 选中样式使用默认checkbox样式，未实现黄色高亮和对勾
4. 后端`batch_update_attributes`接口未实现跳过已有/不存在属性的逻辑

## 实现方案

### 1. 修改按钮控制逻辑

**修改位置**：`updateBatchButtons()` 函数

- 检查所有选中物品是否属于同一类别
- 按钮名称改为"同类属性编辑"
- 不同类别时禁用按钮

### 2. 修改属性选择器数据源

**修改位置**：`showBatchEditModal()` 函数

- 获取选中物品的共同类别
- 从类别模板加载属性数据
- 使用与添加物品相同的属性选择器样式
- 默认展开所有属性（不折叠）

### 3. 修改选中样式

**修改位置**：属性选择器渲染逻辑

- 选中时添加黄色高亮背景
- 添加对勾图标
- 样式与添加物品时保持一致

### 4. 修改批量更新逻辑（前端）

**修改位置**：`batchUpdateAttrs()` 函数

- 校验是否选中属性值
- 传递物品ID和属性ID到后端

### 5. 修改批量更新逻辑（后端）

**修改位置**：`batch_update_attributes()` 函数

- 添加操作：跳过物品已有的属性
- 移除操作：跳过物品没有的属性

## 测试要点

1. 选择不同类别的物品，按钮应禁用
2. 选择同类物品，按钮可用，显示正确属性
3. 选中属性值应显示黄色高亮和对勾
4. 添加已有属性应跳过，不重复添加
5. 移除不存在的属性应跳过，不报错
6. 未选中属性值时点击按钮应提示错误
