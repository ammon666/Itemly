# 样式问题修复计划

## 需求分析

用户提出了7个样式修改需求：

1. **分类筛选选中时整行变紫色** - 侧边栏分类选中时背景变为紫色
2. **分类前图标改为三条线** - 替换小圆点为三条线图标（与类别管理一致）
3. **退出编辑模式保持视图模式** - 记录进入编辑模式前的视图模式，退出时恢复
4. **移除卡片/列表切换动画** - 提升切换速度
5. **列表和筛选按钮文字改为黑色** - 修复白底看不清的问题
6. **列表模式优化** - 名称、分类、属性同行显示，行高自适应，添加分页
7. **统计图标添加颜色** - 分类数量、属性数量的图标添加紫色

## 修改文件

主要修改文件：`frontend/html/index.html`

## 具体修改方案

### 1. 分类筛选选中时整行变紫色

修改 `.tree-row.selected` 的样式：
- 当前：仅文字变紫色
- 修改后：整行背景变为紫色，文字变白

### 2. 分类前图标改为三条线

替换 `.tree-dot` 为三条线图标：
```svg
<svg width="14" height="14" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2">
  <path d="M4 6h16M4 12h16M4 18h7"/>
</svg>
```

### 3. 退出编辑模式保持视图模式

在 `toggleEditMode()` 函数中：
- 进入编辑模式前记录当前视图模式
- 退出编辑模式时恢复之前的视图模式

### 4. 移除卡片/列表切换动画

移除 `setViewMode()` 中的 `loadItems()` 调用，或添加切换时的即时效果

### 5. 列表和筛选按钮文字改为黑色

修改 `.sort-btn` 的颜色为黑色

### 6. 列表模式优化

修改列表项布局：
- 名称、分类、属性同行显示
- 添加分页功能（每页10条）

### 7. 统计图标添加颜色

为分类数量和属性数量的 `stat-ico` 添加 `hi` 类

## 实现步骤

1. 修改 CSS 样式（样式部分）
2. 修改侧边栏分类渲染函数 `renderSidebarCategories()`
3. 修改 `toggleEditMode()` 函数
4. 修改 `setViewMode()` 函数
5. 修改列表项渲染 `updateListView()` 函数
6. 添加分页功能
7. 修改统计卡片渲染

## 风险评估

- 低风险：样式修改不会影响核心功能
- 中等风险：分页功能需要确保与现有代码兼容
