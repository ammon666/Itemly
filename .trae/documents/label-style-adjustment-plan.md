# 标签样式调整计划

## 修改需求

根据用户需求，需要对物品列表的标签样式进行以下调整：

| 序号 | 需求描述 | 修改位置 |
|:---:|---------|---------|
| 1 | 分类标签颜色改为淡紫色 | CSS `.tag-cat` 样式 |
| 2 | 分类标签后面增加分割线 | JavaScript 渲染逻辑 |
| 3 | 分类和属性标签文字改为黑色 | CSS `.tag-cat` 和 `.tag-attr` 样式 |
| 4 | 属性标签背景改为淡黄色 | CSS `.tag-attr` 样式 |
| 5 | 备注标签改为灰底 | CSS/JS 备注标签样式 |
| 6 | 列表视图增加自动递增序号列 | JavaScript `updateListView` 函数 |

## 修改方案

### 1. CSS 样式修改

**文件**: `frontend/html/index.html`

**修改内容**:
- `.tag-cat` 样式：背景改为淡紫色（#EDE9FE），文字改为黑色（#18181B）
- `.tag-attr` 样式：背景改为淡黄色（#FEFCE8），文字改为黑色（#18181B）
- `.tag-remark` 样式：新增，背景为灰色（#F3F4F6），文字为黑色

### 2. JavaScript 渲染逻辑修改

**文件**: `frontend/html/index.html`

**修改内容**:
- `renderItems()` 函数（卡片视图）：分类标签后添加分割线
- `updateListView()` 函数（列表视图）：增加序号列，分类标签后添加分割线

## 修改步骤

1. 修改 CSS 中 `.tag-cat` 和 `.tag-attr` 的样式定义
2. 添加 `.tag-remark` 新样式
3. 添加 `.tag-divider` 分割线样式
4. 修改卡片视图的标签渲染逻辑
5. 修改列表视图的渲染逻辑，增加序号列

## 风险评估

- 低风险：仅修改样式和渲染逻辑，不影响数据层
- 需要注意：序号是显示层面的自动序号，不与物品属性绑定，分页时需基于全局索引而非当前页索引

## 文件清单

- `frontend/html/index.html` - 唯一需要修改的文件
