# Itemly 产品需求文档

## 1. 产品概述

Itemly 是一款轻量化、简化版的个人物品管理系统，类似于 HomeBox，专为个人/家庭使用设计。系统提供简洁直观的界面，支持物品管理、分类标签管理、模板管理等功能，采用移动端优先的设计理念，让用户能够随时随地管理个人物品。

**核心价值**：极简主义设计，单用户本地部署，数据自主可控。

---

## 2. 功能需求

### 2.1 用户角色

| 角色 | 说明 | 权限 |
|------|------|------|
| 单用户 | 系统唯一用户 | 全部功能 |

### 2.2 功能模块

1. **首页模块** - 展示所有物品、搜索筛选、统计信息
2. **添加物品模块** - 图片上传压缩、模板/自由添加
3. **管理模块（后台）** - 用户管理、物品管理、设置
4. **类别管理模块** - 多层级类别增删改查
5. **标签管理模块** - 多层级标签增删改查
6. **模板管理模块** - 按类别创建模板、绑定自定义字段

---

## 3. 页面详细设计

### 3.1 页面清单

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 登录页 | 用户认证 | 登录系统，默认账号 admin/admin123 |
| 首页 | 物品展示 | 卡片视图展示所有物品，支持筛选搜索 |
| 添加物品页 | 物品创建 | 图片上传压缩、模板选择、字段填写 |
| 物品详情页 | 物品查看 | 查看物品完整信息 |
| 管理后台 | 用户管理 | 修改密码、修改个人信息 |
| 管理后台 | 物品管理 | 查看、编辑、删除、批量操作 |
| 类别管理 | 类别CRUD | 多层级类别树形管理 |
| 标签管理 | 标签CRUD | 多层级标签树形管理 |
| 模板管理 | 模板CRUD | 模板创建、字段绑定 |

### 3.2 首页模块设计

- **布局**：移动端优先，响应式卡片网格
- **顶部**：Logo、搜索框、筛选按钮
- **中部**：物品卡片（图片+名称+类别+标签）
- **底部**：添加按钮（悬浮）、统计入口
- **统计面板**：物品总数、分类统计、标签统计

### 3.3 添加物品模块设计

- **图片区域**：点击上传、自动压缩、预览
- **基本信息**：名称（必填）、类别（必填）
- **扩展信息**：标签（多选）、备注
- **模板支持**：可选择模板自动填充字段

## 4. 核心流程

### 4.1 用户登录流程

```
用户打开系统 → 输入账号密码 → 验证成功 → 进入首页
                              ↓
                         验证失败 → 提示错误信息
```

### 4.2 添加物品流程

```
点击添加按钮 → 选择添加方式（模板/自由）→ 填写信息 → 上传图片（必填）
                                                      ↓
图片压缩处理 → 提交数据 → 保存成功 → 返回首页
```

### 4.3 搜索筛选流程

```
输入关键词/选择类别/选择标签 → 点击搜索 → 显示筛选结果 → 清空筛选 → 显示全部
```

## 5. 技术架构

### 5.1 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | Bootstrap 5 + 原生JS | 响应式UI框架 |
| 前端图片处理 | browser-image-compression | 前端图片压缩 |
| 后端 | Python Flask | 轻量级Web框架 |
| 数据库 | SQLite | 单文件数据库 |
| 部署 | Docker + Docker Compose | 容器化部署 |

### 5.2 项目结构

```
Itemly/
├── backend/              # 后端代码
│   ├── app.py           # Flask应用入口
│   ├── models.py        # 数据库模型
│   ├── routes/          # 路由模块
│   └── utils/           # 工具函数
├── frontend/            # 前端代码
│   ├── assets/         # 静态资源
│   ├── css/            # 样式文件
│   ├── js/             # JavaScript文件
│   └── html/           # HTML页面
├── uploads/             # 上传文件目录
├── docker-compose.yml   # Docker Compose配置
├── Dockerfile          # Docker镜像配置
└── README.md           # 项目说明
```

## 6. 数据库设计

### 6.1 数据模型

| 表名 | 说明 | 主要字段 |
|------|------|----------|
| users | 用户表 | id, username, password_hash |
| categories | 类别表 | id, name, parent_id, sort_order |
| tags | 标签表 | id, name, parent_id, sort_order |
| templates | 模板表 | id, name, category_id |
| template_fields | 模板字段表 | id, template_id, field_name, field_type |

## 8. 视觉设计

### 8.1 设计风格

- **主题**：现代简约、卡片式设计
- **配色**：主色蓝色系，扁平化设计
- **圆角**：中等圆角（8px）
- **阴影**：柔和阴影增加层次感

### 8.2 色彩方案

| 用途 | 颜色 |
|------|------|
| 主色 | #0d6efd (Bootstrap Primary) |
| 成功 | #198754 |
| 警告 | #ffc107 |
| 危险 | #dc3545 |
| 背景 | #f8f9fa |
| 文字 | #212529 |

### 8.3 字体

- **标题**：系统默认无衬线字体
- **正文**：系统默认无衬线字体
- **大小**：Bootstrap 默认比例

### 8.4 响应式断点

| 设备 | 宽度 |
|------|------|
| 手机 | < 576px |
| 平板 | 576px - 768px |
| 桌面 | > 768px |
